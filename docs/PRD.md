# PRD - 台中大眾運輸交通整合 APP

## 1. 專案概述

### 1.1 背景與動機
在日常生活中，我們需要搭乘大眾運輸工具時，每次在連接轉乘都會花費大量時間。例如搭乘捷運要轉公車時，常常無法很準確地定位公車的即時位置。為了解決轉乘等待的焦慮與不便，我們希望透過本 APP 來節省通勤時間，讓大眾運輸動線更加透明且易於掌握。

### 1.2 目標用戶
台中市民，或來台中需要搭乘大眾運輸的民眾。

### 1.3 核心價值主張
提供一站式、跨運具（公車、捷運等）的即時動態查詢與最佳化轉乘路徑建議，不僅只有時刻表，更有準確的即時定位與到站預估，讓通勤族節省不必要的等候時間、精準規劃行程。

---

## 2. 功能需求

以下列出本系統的主要功能與對應的使用者故事：

### 2.1 即時動態資訊整合（核心）
- **作為** 一名通勤族，**我希望** 能在同一個介面看到公車與捷運的即時到站位置與班次時間，**以便** 精準掌握出門與轉乘的時機。
- *技術細節*：串接 TDX API 獲取台中公車與捷運動態數據，實作後端 Flask 路由與前端自動重新整理機制。

### 2.2 一站式平台查詢
- **作為** 一個外地旅客，**我希望** 能在一個平台上查詢到公車、捷運等多種大眾運輸工具的資訊，**以便** 不用在不同 APP 之間切換。

### 2.3 轉乘時間最佳化路徑演算
- **作為** 一個趕時間的乘客，**我希望** 輸入起終點後，系統能自動計算結合「捷運 + 公車」等組合的最佳路徑並提供 ETA，**以便** 我能選擇最快抵達的路線。

### 2.4 多運具即時動態整合看板
- **作為** 一名準備轉乘的乘客，**我希望** 在地圖上點擊特定轉乘站點時，能同時看到「公車進站倒數」與「捷運發車時刻」的清單，**以便** 我能快速做決定。

### 2.5 常用路線與轉乘點收藏
- **作為** 一名日常通勤的大學生，**我希望** 能一鍵收藏我每天上下學的路線與站點，**以便** 我下次打開 APP 時能直接查看，減少重複搜尋的時間。

### 2.6 站點擁擠度顯示
- **作為** 一名不喜歡擁擠的乘客，**我希望** 能在 APP 上看到公車或捷運的即時擁擠程度（如：紅-擁擠、綠-通暢），**以便** 我評估是否要等下一班車。

## 3. 非功能需求

### 3.1 技術限制
- **前端**：HTML / CSS / JavaScript（純前端，無框架，遵循響應式網頁設計）。
- **後端**：Python + Flask。
- **資料庫**：SQLite，用於儲存使用者的「常用路線」等偏好設定。
- **外部資源**：TDX API（交通運輸資料流通服務平臺）、Leaflet.js（輕量化互動地圖）。

### 3.2 效能與安全考量
- **數據延遲容錯**：需實作快取機制與本地暫存，當 TDX API 失效或延遲時，能顯示最後一筆有效資訊並標記「離線」狀態。
- **重新整理機制**：在地圖上明確標示資料更新時間，並允許前端介面手動/自動觸發最新位置請求，確保及時定位的精確性。

---

## 4. MVP 範圍

### Must Have（核心必須功能）
- 即時交通數據整合模組（串接 TDX API，顯示公車/捷運動態）。
- 一站式平台路線查詢（輸入起迄點）。
- 互動式地圖呈現（使用 Leaflet.js 標記站牌與車輛位置）。

### Should Have（重要但不影響核心運作）
- 多運具轉乘路徑演算（捷運+公車最佳化組合）。
- 常用路線與轉乘點「一鍵收藏」功能。

### Nice to Have（加分項目）
- 試算時間與價格功能（即時計算總花費）。
- 站點狀態與擁擠度顯示（視覺化指標）。

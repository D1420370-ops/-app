# 產品需求文件 (PRD)

**專案名稱**：台中大眾運輸交通整合 APP
**專案描述**：一款專為台中地區設計的交通整合應用程式，提供精準的各類大眾運輸工具（如捷運、公車等）即時定位與到站時間，並支援路線規劃與票價試算，旨在節省使用者等候時間，減少轉乘焦慮。

## 1. 專案概述

### 背景與動機
在日常生活中，搭乘大眾運輸工具（如捷運轉公車）的轉乘過程常因無法準確掌握下一班車的定位與到站時間，而耗費大量時間與心力。為了節省等候時間、消除等不到車的焦慮，並讓民眾更清楚大眾運輸動線，我們希望透過這款 APP 進而提升大眾運輸的使用體驗。

### 目標用戶
- 台中市民。
- 來台中旅遊、出差，需要搭乘大眾運輸的民眾。

### 核心價值主張
提供「準確到站時間」而非僅有靜態時刻表，結合「精準定位」與「動線檢視」，讓使用者的行程規劃更加明確，大幅減少不必要的等待時間與焦慮感。

## 2. 功能需求

- **功能一：即時動態與時程準確性**
  - 作為 **大眾運輸乘客**，我希望 **能夠看到公車或捷運精準的即時到站時間**，以便 **減少在車站空等的焦慮與時間浪費**。
- **功能二：大眾運輸工具檢視圖與定位精準化**
  - 作為 **使用者**，我希望 **能在地圖上看到車輛的即時精準定位與行駛動線**，以便 **準確掌握交通工具目前的實際位置**。
- **功能三：跨運具路線規劃與替代方案**
  - 作為 **正在轉乘的乘客**，我希望 **系統能提供包含不同運輸工具（如捷運轉公車）的最佳路線規劃及替代方案**，以便 **在突發狀況下也能順利到達目的地**。
- **功能四：旅程時間預估（到達目的地所需時間）**
  - 作為 **行程規劃者**，我希望 **輸入起訖點後能看到預估的總行車與步行時間**，以便 **更好安排接下來的行程**。
- **功能五：票價推算**
  - 作為 **精打細算的通勤族**，我希望 **系統能在規劃路線時一併計算出預估的交通費用**，以便 **控制交通預算**。
- **功能六：台中地區專屬資訊整合**
  - 作為 **使用者**，我希望 **系統能高度整合台中地區的所有交通資訊**，以便 **在台中地區規劃行程時有最精準在地化的體驗**。

## 3. 非功能需求

### 技術限制
- 後端：採用 Flask 框架。
- 前端：使用 Jinja2 模板引擎進行頁面渲染。
- 資料庫：使用 SQLite 以進行輕量化資料儲存。

### 效能考量
- 系統需具備處理高頻率位置更新（如公車 GPS）的串接能力，確保地圖上的定位資訊能即時刷新。
- 路線規劃與時間預估的計算需在使用者可接受的極短時間內完成回傳。

### 安全考量
- 若有使用者個人資料（如常用路線、收藏站點）需妥善保存，確保資料不被未授權存取。
- 第三方 API 串接（如政府 PTX 平台資料）需採用安全的 HTTPS 協定。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須有)
- 大眾運輸工具（公車、捷運）時刻表與即時到站時間查詢。
- 點對點基本路線規劃與預估到達時間。
- 基礎的大眾運輸動線與站點地圖檢視。

### Should Have (應該有)
- 車輛即時精準定位與動態地圖顯示。
- 替代路線方案建議。
- 票價推算功能。

### Nice to Have (可以有)
- 結合周邊景點、YouBike 站點資訊。
- 擁擠度預測與即時天氣提醒。
- 使用者個人帳號系統（收藏站點、常用路線設定）。
